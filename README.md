# Developer Growth Challenge - 30 Days

- This repository documents my 30-day journey to become a better and more practical developer. The goal of this challenge is not to rush through tutorials or collect certificates. Instead, I want to focus on building, practicing, solving problems, and developing the habits of a real developer.

## For the next 30 days:

* I will learn by building projects and solving problems.
* I will push my progress to GitHub every day.
* Each day will have its own folder containing code, notes, and learnings.
* I will maintain a README to track what I learned, what challenges I faced, and how I solved them.
* At the end of the challenge, I will reflect on my growth and compare where I started with where I finished.

## 🎯 Purpose

### I believe that becoming a developer is not about watching endless tutorials.  It is about: 


1. Writing code consistently
2. Building projects
3. Debugging errors
4. Improving problem-solving skills
5. Learning in public
6. Documenting the journey
